# Module 13: Interview Prep

## Overview

Hands-on coding challenges, system design discussions, behavioral scenarios, and common interview patterns for DevOps/SRE roles.

## Subdirectories

| Directory                    | Description               | Key Files                                                                   |
| ---------------------------- | ------------------------- | --------------------------------------------------------------------------- |
| `coding-challenges/`         | 5 practical problems      | Rate limiter, log parser, load balancer, service discovery, circuit breaker |
| `system-design/`             | System design discussions | Monitoring, CI/CD, orchestration, HA                                        |
| `behavioral-questions/`      | STAR-format stories       | Incidents, scaling, collaboration                                           |
| `common-interview-patterns/` | Code patterns             | Error handling, concurrency, API design, optimization                       |

## How to Use

1. **Coding challenges**: Read `problem.md`, attempt `solution.py`, verify with tests, review `explanation.md`
2. **System design**: Use as discussion frameworks for whiteboard interviews
3. **Behavioral**: Adapt with your own experiences using the STAR method
4. **Patterns**: Study and modify the code examples

## Key Interview Topics

1. **Rate limiting** — Token bucket, sliding window algorithms
2. **System design** — Scalability, reliability, observability
3. **Concurrency** — Threading, asyncio, race conditions
4. **Error handling** — Retry patterns, circuit breakers, graceful degradation
5. **Behavioral** — STAR method, blameless culture, incident response
